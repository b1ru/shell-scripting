Η εντολή exec

Prerequisites

Subshells
I/O Redirection

Η εντολή exec έχει δύο χρήσεις αναλόγως αν χρησιμοποιείται με arguments ή όχι.

Με arguments

exec <command>

Θα τρέξει το <command> στο τρέχων shell, αντικαθιστώντας το πρόγραμμα που έτρεχε ήδη στο τρέχων shell.
Δεν θα δημιουργηθεί δηλαδή subshell (καινούριο process).

Συνέπειες

  • Ο έλεγχος δεν θα επιστρέψει ποτέ στο πρόγραμμα που κάλεσε το exec.

  • Αν δηλαδή το πρόγραμμα που κάλεσε το exec έχει γραμμές κάτω από την γραμμή που καλεί το exec, τότε αυτές δεν θα εκτελεστούν ποτέ.

Παράδειγμα

  1. Δημιουργήστε τα παρακάτω προγράμματα στον υπολογιστή σας:

    program-1.sh
    #!/bin/sh

ls (2)
echo "PROGRAM 1 DONE" (3)
    program-2.sh
    #!/bin/sh

exec ls (2)
echo "PROGRAM 2 DONE" (3)

  2. Κάντε τα εκτελέσιμα:

    $ chmod u+x program-1.sh program-2.sh

  3. Εκτελέστε το πρώτο πρόγραμμα. Τι θα τυπώσει?

    $ ./program-1.sh (1)
    program-1.sh
    #!/bin/sh

ls (2)
echo "PROGRAM 1 DONE" (3)
    1 Το shell δημιουργεί ένα subshell για να τρέξει το program-1.sh.
    O έλεγχος περνάει στο program-1.sh subshell.
    Αρχίζει η εκτέλεση του program-1.sh.
    2 To program-1.sh shell δημιουργεί ένα subshell για να τρέξει την εντολή ls.
    Ο έλεγχος πάει στο ls subshell.
    Εκτελείται η εντολή ls και τυπώνει τα περιεχόμενα του working directory.
    Αφού τελειώσει η εκτέλεση του ls, το ls subshell παύει να υπάρχει και ο έλεγχος επιστρέφει στο program-1.sh shell.
    3 To program-1.sh shell δημιουργεί ένα subshell για να τρέξει την εντολή echo.
    O έλεγχος πάει στο echo subshell.
    Εκτελείται η εντολή echo και το μήνυμα PROGRAM 1 DONE εμφανίζεται στην οθόνη.
    Αφού τελειώσει η εκτέλεση του echo το echo subshell παύει να υπάρχει και ο έλεγχος επιστρέφει στο program-1.sh shell.
    Επειδή η echo είναι η τελευταία εντολή του program-1.sh, το program-1.sh subshell παύει να υπάρχει και ο έλεγχος επιστρέφει στο αρχικό shell, στο οποίο μπορούμε τώρα να τρέξουμε νέες εντολές.

  4. Εκτελέστε το δεύτερο πρόγραμμα. Τι θα τυπώσει?

    $ ./program-2.sh (1)
    program-2.sh
    #!/bin/sh

exec ls (2)
echo "PROGRAM 2 DONE" (3)
    1 Το shell δημιουργεί ένα subshell για να τρέξει το program-2.sh.
    O έλεγχος περνάει στο program-2.sh subshell.
    Αρχίζει η εκτέλεση του program-2.sh.
    2 Το πρόγραμμα ls θα αντικαταστήσει το program-2.sh στο τρέχων shell (program-2.sh subshell).
    Εκτελείται η εντολή ls και τυπώνει τα περιεχόμενα του working directory.
    Αφού τελειώσει η εκτέλεση του ls, το program-2.sh subshell παύει να υπάρχει και ο έλεγχος επιστρέφει στο αρχικό shell, στο οποοίο μπορούμε τώρα να τρέξουμε νέες εντολές.
    3 Η εντολή echo δεν θα εκτελεστεί ποτέ καθώς η εντολή ls αντικατέστησε το πρόγραμμα program-2.sh, οπότε ο έλεγχος δεν θα ξαναγυρίσει ποτέ σε αυτό.

Χωρίς arguments

Αν το τρέξουμε χωρίς arguments, κάνει redirect το standard input ή το standard output.

Το redirection έχει ισχύ μέχρι να ξαναχρησιμοποιηθεί κάποια exec ή μέχρι να τερματιστεί το shell.
exec < file

Κάνε redirect το standard input στο file.

exec > file

Κάνε redirect το standard output στο file.
